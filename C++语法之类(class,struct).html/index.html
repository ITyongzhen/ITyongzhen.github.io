指针访问对象成员的本质

下面代码最后打印出来的每个成员变量值是多少？

第一感觉是

id = 40 age = 50 height = 30

然后用编译器运行起来的结果是

id = 10 age = 40 height = 50

原因

  • 因为指针p实际指向的是person.age的地址，但是(Person *)&person.age; 欺骗编译器是指向person。
  • 64位下person中每个成员变量都是Int类型，占用4个字节。
  • 当我们修改p->id = 40;的时候，实际上修改的是age指向的内存空间。
